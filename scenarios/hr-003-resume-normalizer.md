# Нормализация резюме в единый формат

## Мета

| Поле | Значение |
|---|---|
| **ID** | `hr-003` |
| **Статус** | `verified` |
| **Функция** | `hr` |
| **Отрасль** | `universal` |
| **Размер компании** | `small`, `medium`, `large` |
| **Сложность** | `no-code` |
| **Бюджет** | `free` |
| **Срок до результата** | `days` |

## Боль

Резюме приходят в десятках форматов (PDF, DOC, RTF, ссылки) с разной структурой. HR тратит часы на ручное приведение к единому виду перед передачей нанимающему менеджеру или заказчику. При потоке 20–50 резюме в неделю это становится основной работой.

## Решение

Бот принимает резюме в любом формате или ссылку (Google Doc, HH.ru), извлекает структурированные данные и создаёт унифицированный документ по заданному шаблону компании. HR получает готовый файл за секунды.

## Шаги внедрения

1. **Шаблон** — зафиксировать стандарт резюме: какие поля, в каком порядке, в каком формате
2. **Бот-приёмник** — Telegram-бот или email-парсер принимает файлы/ссылки
3. **Извлечение данных** — ИИ парсит резюме и структурирует по заданным полям
4. **Генерация документа** — автоматическое создание Google Doc по шаблону
5. **Доставка** — готовый документ отправляется HR или сохраняется в папку

## Инструменты

| Инструмент | Роль |
|---|---|
| Claude / GPT-4 | Извлечение и структурирование данных |
| Telegram Bot / Email | Приём резюме |
| n8n / Make | Оркестрация пайплайна |
| Google Docs API | Генерация итогового документа |

## Подводные камни

- Резюме с нестандартной вёрсткой (таблицы, колонки) могут парситься с ошибками
- Сканированные PDF без текстового слоя требуют OCR
- Нужна проверка: ИИ иногда «галлюцинирует» поля, которых нет в оригинале
- Для ссылок на HH.ru могут быть ограничения на автоматический парсинг
