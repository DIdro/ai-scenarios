## Боль

Анализ одного тендерного лота занимает 80–160 человеко-часов. Ошибки в оценке сроков и стоимости достигают 20–30%. Пока компания считает — конкуренты уже подали заявку.

## Решение

Система получает исходно-технические требования (ИТТ), автоматически разбирает их структуру, сопоставляет с базой выполненных проектов, оценивает трудоёмкость по аналогам и формирует параметрическую ценовую модель.

## Шаги внедрения

1. **База проектов** — структурировать и загрузить выполненные проекты с фактической трудоёмкостью
2. **Парсер ИТТ** — ИИ извлекает ключевые параметры из документации (типы работ, объёмы, требования)
3. **Модуль сопоставления** — поиск аналогичных проектов в базе, расчёт коэффициентов
4. **Ценовая модель** — параметрический расчёт на основе исторических данных
5. **Языковой интерфейс** — менеджер задаёт вопросы к документации на естественном языке

## Инструменты

| Инструмент | Роль |
|---|---|
| Claude / GPT-4 | Разбор ИТТ, языковой интерфейс |

## Подводные камни

- Требует структурированной базы прошлых проектов — если данных нет, система неточна
- Специфика отрасли: хорошо работает для типовых изделий, хуже для уникальных
- Юридические риски: итоговое КП всё равно должен проверить человек
- Долгое накопление данных для калибровки модели
