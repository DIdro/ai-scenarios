# Автоматическая обработка протоколов совещаний

## Мета

| Поле | Значение |
|---|---|
| **ID** | `ops-001` |
| **Статус** | `verified` |

| **Бюджет** | `low` |
| **Срок до результата** | `weeks` |

## Боль

На оперативных совещаниях обсуждается 12–40 вопросов. Часть теряется, часть «зависает» без ответственного. Ручная расшифровка и структурирование протокола занимают 1–3 часа после каждой встречи. Задачи не попадают в трекер.

## Решение

ИИ получает транскрипт встречи (или фото с доски/флипчарта), извлекает вопросы и решения, категоризирует их, находит дубликаты и автоматически создаёт задачи в системе управления проектами с назначением ответственных.

## Шаги внедрения

1. **Настройка транскрибации** — подключить сервис транскрибации к формату встреч (Zoom/Teams/запись на телефон)
2. **Промпт для извлечения** — разработать промпт, который извлекает: вопросы, решения, ответственных, дедлайны
3. **Дедупликация** — добавить шаг сравнения с открытыми задачами в трекере
4. **Интеграция с трекером** — через API или n8n автоматически создавать задачи
5. **Граф взаимосвязей** — опционально: строить связи между задачами (зависимости)

## Инструменты

| Инструмент | Роль |
|---|---|
| MyMeet.ai / Whisper | Транскрибация аудио |
| Claude / GPT-4 | Извлечение структуры, дедупликация |
| n8n / Make | Оркестрация пайплайна |
| Яндекс.Трекер / Jira / АтласТрекер | Создание задач |

## Результаты

| Метрика | До | После |
|---|---|---|
| Время на протокол | 1–3 часа | 10–15 минут |
| % совещаний с зависшими задачами | ~60% | ~15% |
| Время встреч | базовое | −20–30% |

## Подводные камни

- Качество транскрипции падает при плохом звуке или нескольких говорящих одновременно
- Требует дисциплины команды: встречи нужно записывать
- Автоматическое назначение ответственных работает только если имена совпадают с профилями в трекере
- Первые 2–3 недели нужна ручная проверка перед отправкой задач
