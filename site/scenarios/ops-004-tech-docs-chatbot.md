## Триггер

Сотрудник производства задаёт вопрос по документации (чертёж, спецификация, регламент).

## Вход

Вопрос на естественном языке + корпус технической документации (PDF, DOC, чертежи с описаниями).

## Выход

Ответ с точной цитатой и ссылкой на документ: «Согласно чертежу ДТ-2045 (стр. 3), допуск составляет ±0.05 мм».

## Боль

Сотрудники производства постоянно обращаются к конструкторам с вопросами по документации. Эксперты тратят часы на рутинные консультации вместо основной работы.

## Решение

RAG-система на базе всей технической документации. Сотрудник задаёт вопрос в чате — бот находит нужный раздел и отвечает с цитатой. Эксперты подключаются только к нетиповым вопросам.

## Шаги внедрения

1. **Инвентаризация** — собрать актуальные документы (PDF, DOC, описания к чертежам)
2. **Векторизация** — загрузить документы в векторную базу данных
3. **RAG-пайплайн** — поиск + генерация ответа с цитатами
4. **Интерфейс** — Telegram-бот или веб-чат
5. **Эскалация** — кнопка «задать эксперту» для нетиповых вопросов

## Подводные камни

- Работает только с актуальной документацией — устаревшие файлы дают неверные ответы
- Чертежи без текстового описания плохо индексируются
- Требует регулярного обновления базы при изменении документации
