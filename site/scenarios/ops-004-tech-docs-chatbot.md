## Боль

Сотрудники производства постоянно обращаются к конструкторам и технологам с вопросами по документации — чертежам, спецификациям, регламентам. Эксперты тратят часы на рутинные консультации вместо основной работы.

## Решение

RAG-система на базе всей конструкторской и технологической документации компании. Сотрудник задаёт вопрос в чате — бот находит нужный раздел документации и отвечает с точными ссылками на источник. Эксперты подключаются только к нетиповым вопросам.

## Шаги внедрения

1. **Инвентаризация документации** — собрать все актуальные документы (PDF, DOC, чертежи с описаниями)
2. **Векторизация** — загрузить документы в векторную базу данных
3. **RAG-пайплайн** — настроить поиск по документации + генерацию ответа с цитатами
4. **Интерфейс** — Telegram-бот или веб-чат, интегрированный в рабочее место
5. **Обратная связь** — кнопка «эксперт» для эскалации нетиповых вопросов

## Подводные камни

- Работает только с актуальной документацией — устаревшие файлы дают неверные ответы
- Чертежи без текстового описания плохо индексируются — нужна предобработка
- Требует регулярного обновления базы при изменении документации
- Сотрудники должны доверять системе — первое время нужна верификация ответов экспертами
