# Онтология сценариев AI Scenarios Library v1.0

## Принцип

Сценарий — это воспроизводимый паттерн применения ИИ в бизнесе.
Не кейс конкретной компании, а класс ситуаций с конкретным входом и выходом.

Онтология — гетерархия: ни одно измерение не главное.
Сценарий описывается пересечением значений из нескольких независимых фасетов.

---

## Обязательные фасеты (поиск и фильтрация)

### ai_pattern — что делает ИИ
| Значение | Описание |
|---|---|
| `rag` | Поиск по документам и генерация ответа на основе найденного |
| `nql` | Запросы к данным на естественном языке |
| `classification` | Классификация, маршрутизация, категоризация входящего |
| `generation` | Генерация документов, текстов, кода по шаблону или описанию |
| `agent` | Автономное выполнение цепочки действий по триггеру |
| `extraction` | Извлечение структурированных данных из неструктурированного входа |
| `monitoring` | Наблюдение, алертинг, дайджест по данным из источников |

### function — в каком отделе
`sales` / `hr` / `operations` / `finance` / `it` / `marketing` / `management` / `legal` / `rd`

### industry — в каком типе бизнеса воспроизводимо
`universal` / `manufacturing` / `services` / `retail` / `it-company` / `education` / `healthcare` / `fintech`

### autonomy — насколько ИИ действует самостоятельно
| Значение | Описание |
|---|---|
| `chat` | Человек спрашивает → ИИ отвечает. Без интеграций и триггеров |
| `cowork` | ИИ встроен в процесс, человек проверяет и утверждает результат |
| `agent` | ИИ запускается по триггеру, выполняет цепочку, результат без человека |
| `multi-agent` | Несколько агентов координируются, каждый отвечает за свою часть |

### process_maturity — что нужно иметь до внедрения
| Значение | Описание |
|---|---|
| `chaos` | Процесса нет, всё в головах людей |
| `process-exists` | Процесс есть, но не автоматизирован |
| `data-exists` | Данные накоплены, не используются |

### effect — что улучшается (можно несколько)
`time-saving` / `revenue-growth` / `error-reduction` / `scaling` / `expert-independence`

## Поля карточки (описательные, не фильтруются)

| Поле | Значения |
|---|---|
| `scale` | `micro` (одна операция) / `meso` (один процесс) / `macro` (сквозной) |
| `input_type` | `text` / `documents` / `audio` / `structured-data` / `api` / `mixed` |
| `complexity` | `no-code` / `low-code` / `custom-dev` |
| `budget` | `free` / `low` / `medium` / `high` |
| `time_to_result` | `days` / `weeks` / `months` |

---

## Критерий валидности сценария

Сценарий принимается если:
1. ✅ Есть конкретный **триггер** — что запускает сценарий
2. ✅ Есть конкретный **вход** — что подаётся ИИ
3. ✅ Есть конкретный **выход** — что получается на выходе
4. ✅ **Воспроизводимо** в классе похожих компаний/ситуаций
5. ❌ Если «зависит от...» встречается больше 2 раз — сценарий нужно дробить

## Конвейер добавления сценария

```
Идея → Проверка критериев → Определение фасетов → Написание карточки → Ревью → Публикация
```

При ревью проверяется:
- Уровень гранулярности совпадает с другими сценариями той же функции
- Все обязательные фасеты заполнены
- Триггер/вход/выход конкретны
