## Что болит

Анализ одного тендерного лота занимает 80–160 человеко-часов. Ошибки
в оценке сроков и стоимости достигают 20–30%. Пока компания считает —
конкуренты уже подали заявку.

## Как это работает

Система получает исходно-технические требования (ИТТ), автоматически
разбирает их структуру, сопоставляет с базой выполненных проектов,
оценивает трудоёмкость по аналогам и формирует параметрическую
ценовую модель.

<Steps>
1. **База проектов.** Структурируем и загружаем выполненные проекты
   с фактической трудоёмкостью.
2. **Парсер ИТТ.** ИИ извлекает из документации ключевые параметры:
   типы работ, объёмы, требования.
3. **Модуль сопоставления.** Поиск аналогичных проектов в базе и
   расчёт коэффициентов.
4. **Ценовая модель.** Параметрический расчёт на основе исторических
   данных.
5. **Языковой интерфейс.** Менеджер задаёт вопросы к документации
   на естественном языке.
</Steps>

## Подводные камни

<Callout type="warning">
  Без структурированной базы прошлых проектов система неточна.
  Если данных нет — сценарий начинается со сбора и оцифровки истории,
  а не с автоматизации.
</Callout>

Хорошо работает для типовых изделий, хуже для уникальных. Юридически
итоговое КП всё равно должен проверять человек — снять ответственность
с системы не получится.
