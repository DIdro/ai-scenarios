# Как добавить сценарий

Спасибо, что хотите пополнить библиотеку! Добавление сценария —
**один MDX-файл + Pull Request**. Править код не нужно.

> Подробный шаблон, дерево и список отраслей — в [README.md](README.md#как-добавить-сценарий).

## Шаги

1. **Форк** репозитория (или ветка, если вы коллаборатор).
2. Выберите узел дерева в [`site/lib/tree.ts`](site/lib/tree.ts): `subsystem` → `process` → `subprocess`.
3. Создайте файл `site/content/<subsystem>/<process>/<slug>.mdx`.
4. Заполните frontmatter (`title`, `description`, `author`, `authorOrg`, `publishedAt`,
   `subprocess`, `industries`, `forWhom`) — см. пример в README.
5. Напишите статью по единому шаблону:
   Лид → `## Что болит` (+ `<Stat>`) → `## Что делает ИИ` (+ `<Steps>`) →
   `## Что компания получает` → `## Где нужен человек` (+ `<Callout type="warning">`).
6. Проверьте локально: `cd site && npm run build`.
7. Откройте **Pull Request**. Мы рассмотрим и смёрджим.

## Правила

- **Только реальный опыт** — никаких гипотетических или сгенерированных сценариев.
- Один сценарий = один узел дерева; кросс-срез — через теги отраслей (`industries`).
- Честный блок «Где нужен человек» обязателен — без маркетингового «ИИ решит всё».
- У каждой статьи есть автор.

## Без агента/GitHub

Можно просто описать кейс через форму «Предложить сценарий» на сайте
[scenarios.diip-method.ru](https://scenarios.diip-method.ru/) — мы оформим сами.
